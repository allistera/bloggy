# Kubernetes Chaos Engineering

Chaos engineering is critical to establishing trust in Kubernetes clusters.

<Callout type="tip" title="Start small">
  Inject one fault class at a time (for example network delay only) before combining
  latency, stress, and I/O chaos in the same experiment.
</Callout>

## Why Chaos Mesh?

Chaos Mesh provides full lifecycle fault injection capabilities:
- Network latency injection
- CPU/Memory stress testing
- Disk I/O failures

Here is an example chaos configuration:

```yaml
apiVersion: chaos-mesh.org/v1alpha1
kind: NetworkChaos
metadata:
  name: network-delay
spec:
  action: delay
  mode: one
  selector:
    namespaces:
      - default
  delay:
    latency: '100ms'
```

Running tests against this ensures that services degrade gracefully rather than collapsing entirely.
